|---|---|---|---|
| 单独目录，不阅读和依赖原杂项内容 | 项目限定在 `paper-agent/`，规划独立 Git 仓库 | TASKS M0 | 仓库依赖、测试夹具和构建过程不引用父目录文件 |

| 自动按工作流开始写作 | `run` 编排 Skill、规划、写作、审计和导出 | TASKS M6 | 一条命令从案例与资料生成草稿和审计报告 |

| 不伪造来源、数据和调研过程 | 来源台账、Evidence ID、真实性门禁和未核验占位 | TASKS M4、M5 | 假 DOI、假数据和缺失调研材料的故障测试被阻断 |
| 之后发布到 GitHub | CI、Apache-2.0、README、贡献/安全文档、示例和可构建 wheel | TASKS M0、M8 | wheel 隔离安装并从示例完成全链；远程创建与推送等待用户授权 |

## 当前一致性结论

v0.1.0 的实现覆盖全部明确首发需求。自动化与安全边界的关系是：正常高置信路径不要求人工确认；只有输入冲突、证据不足、疑似虚构或格式无法可靠解析等阻断级问题才暂停。正式跨学科效果基准、OCR、数据库检索和学校模板适配仍属于后续路线图，不能作为当前能力宣传。

# 一次生成成功率评测协议

更新日期：2026-07-16

## 目标

评测对象是“一次 `paper-agent run` 能否在不人工改写中间产物的情况下，生成达到严格门禁的高完成度草稿”。它不是分数预测，也不把同一个模型对自身文本的评价当作唯一证据。

## 最小案例矩阵

每个发布候选至少包含以下四类任务，每类至少三个不同主题：

| 体裁 | 必备输入 | 特别检查 |
|---|---|---|
| 通识课结课论文 | 任务书、3–5 篇同类案例、3 个以上来源 | 可争辩论点、反方观点、论证担保 |
| 文献综述 | 任务书、3–5 篇案例、5 个以上来源 | 跨来源主题综合、共识/分歧、证据评价 |
| 课程调研报告 | 任务书、3–5 篇案例、来源和真实 data | 方法—结果分离、数字证据、限制、建议可追溯 |
| 本科毕业论文 | 任务书、3–5 篇案例、来源/真实材料 | 问题—理论—方法/材料—分析—讨论闭环 |

案例不得从项目训练提示中直接复制。任务书要包含至少一个格式硬约束、一个内容硬约束和一个禁止事项，以验证要求优先级。

## 单次判定

机器门禁直接读取 `audit.json`：

- `passed=true`：无确定性 blocker，仅表示没有发现假 Evidence ID、缺节等阻断问题。
- `one_shot_success=true`：进一步要求无 important、篇幅和分节完成、无占位与无证据数字，并且六个独立质量维度都不低于 70。

人工盲评另用同一六维量表，每维 0–100，并追加两项二元检查：是否存在无法由来源支持的关键事实；是否存在明显照搬案例。每份草稿至少由两名评阅者独立评价，分歧超过 15 分时由第三人复核。

## 多次试验与报告

模型输出有随机性。每个“模型版本 × 案例”至少运行三次，并保存完整配置、模型精确名称、温度、输入哈希、Skill 版本、`outline.json`、`audit.json` 和耗时。报告以下指标：

1. 严格一次成功率：`one_shot_success=true` 的 trial 比例。
2. 安全通过率：无 blocker 的 trial 比例。
3. 六维得分的中位数和四分位距，不只报告平均值。
4. 失败原因分布：任务遗漏、篇幅不足、证据不足、推理薄弱、体裁动作缺失、连贯性、引用问题、供应商错误。
5. 成本和延迟：总请求数、重试数、输入/输出 token（供应商可提供时）和墙钟时间。

不同模型必须使用相同输入、温度和最大重试。若模型上下文限制不同，需要在报告中单列，不能静默截断资料。

## 当前自动化覆盖与仍需完成的证据

自动测试已经覆盖四类体裁合同、证据 ID 校验、篇幅/分节门禁、六维评分阈值和供应商瞬态重试。测试中的高分 evaluator 是固定回放，只证明判定器按合同工作，不证明任何真实模型达到该质量。

公开“成功率”数字之前，必须用真实任务材料执行上述重复试验，并进行人工盲评。项目不会从单次演示、离线 deterministic 文本或模型自评推导成功率。
